# Short Pad Attack (Low Exponent + Small Padding)

This attack applies when RSA uses a **small exponent (usually `e = 3`)** and messages are padded with a **small random value**.

```text
M1 = m + r1
M2 = m + r2
```

where:

```text
m  = original message (same)
r1, r2 = small padding values
```

Encryption:

```text
C1 = (m + r1)^e mod n
C2 = (m + r2)^e mod n
```

Goal:

```text
Recover m
```

## 2. Core Idea

The padding is **too small**, so:

```text
Difference between messages is small
```

This allows us to:

```text
Treat it like a related message attack
```

But instead of known relation:

```text
M2 = M1 + k   (known k)
```

we have:

```text
M2 = M1 + (r2 − r1)   (unknown but small)
```

## 3. Mathematical Setup

Let:

```text
x = m
```

Then:

```text
C1 = (x + r1)^e mod n
C2 = (x + r2)^e mod n
```

Define:

```text
Δ = r2 − r1   (small)
```

So:

```text
(x + r2) = (x + r1 + Δ)
```

## 4. Key Insight

```text
Δ is small → brute-force possible
```

Once Δ is guessed:

```text
M2 = M1 + Δ
```

Now it becomes:

```text
Franklin–Reiter attack
```

## 5. Attack Strategy

1. Loop over small Δ
2. Assume:

```text
M2 = M1 + Δ
```

3. Build:

```text
f(x) = x^e − C1
g(x) = (x + Δ)^e − C2
```

4. Compute:

```text
gcd(f, g)
```

5. If valid:

```text
Recover x = m
```

## 6. Python (SageMath) Implementation

```python
# SageMath required

n = ...
e = 3

C1 = ...
C2 = ...

R.<x> = PolynomialRing(Zmod(n))

for delta in range(1, 1000):  # small padding search
    f = x^e - C1
    g = (x + delta)^e - C2

    h = f.gcd(g)

    if h.degree() == 1:
        m = -h[0]
        print("Found m:", m)
        break
```

## 7. Why This Works

```text
Small padding → small Δ
Small Δ → brute-force feasible
After guess → becomes algebraic (Franklin-Reiter)
```

## 8. When to Use

Look for:

```text
Same message encrypted twice
Small random padding
Same n and e
e = 3 (very common)
```

Hints like:

```text
"short padding"
"random suffix"
"tiny noise added"
```

## 9. Important Notes

```text
Works only if padding difference is small
Requires polynomial GCD → use Sage
Fails if padding is large or randomized properly
```

## 10. Example (Concept)

```text
m = 100
r1 = 2 → M1 = 102
r2 = 5 → M2 = 105
```

```text
Δ = 3
```

We brute Δ:

```text
Try Δ = 3 → success
```

## 11. Complexity

```text
O(small_padding_range)
```

Usually:

```text
< 2^16 or smaller
```

## 12. Summary

```text
Same message + small padding
→ guess Δ
→ reduce to Franklin-Reiter
→ recover m
```
