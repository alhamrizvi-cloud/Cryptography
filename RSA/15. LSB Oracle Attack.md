# LSB Oracle Attack (Least Significant Bit Oracle)


This is an **interactive attack**.
You do not get keys or factors. Instead, you get access to an **oracle** that reveals:

```text
The least significant bit (LSB) of the decrypted message
```

Formally:

```text
Oracle(C) → (M mod 2)
```

Goal:

```text
Recover full plaintext M
```

## 2. Problem Setup

RSA:

```text
C = M^e mod n
```

You can query the oracle with any ciphertext `C'` and it returns:

```text
LSB( M' ) where M' = (C')^d mod n
```

## 3. Core Idea

RSA is multiplicative:

```text
(C · k^e) mod n → (M · k) mod n after decryption
```

So if we send:

```text
C' = C · (2^e mod n)
```

Then:

```text
Decryption gives: 2M mod n
```

## 4. Key Insight

By querying:

```text
C · (2^e)^i mod n
```

we get:

```text
2^i · M mod n
```

And the oracle tells:

```text
Is this value even or odd?
```

## 5. What This Gives

Each query gives **one bit of information** about `M`.

We use this to **narrow the interval** where `M` lies.

## 6. Interval Reduction Idea

Start with:

```text
M ∈ [0, n]
```

Each oracle query:

```text
splits the range in half
```

## 7. Attack Logic

At step `i`:

```text
Multiply ciphertext by (2^e)^i
Query oracle
```

If result is:

```text
Even → M is in lower half
Odd  → M is in upper half
```

## 8. Iterative Process

We maintain:

```text
low = 0
high = n
```

Each step:

```text
mid = (low + high) / 2
```

Then:

```text
If oracle says even → high = mid
If oracle says odd  → low = mid
```

## 9. Python Simulation

```python
def lsb_oracle_attack(n, e, C, oracle):
    low = 0
    high = n

    c = C
    multiplier = pow(2, e, n)

    for _ in range(n.bit_length()):
        c = (c * multiplier) % n
        bit = oracle(c)

        mid = (low + high) // 2

        if bit == 0:
            high = mid
        else:
            low = mid

    return low
```

## 10. Oracle Example (Simulation)

```python
def oracle(C):
    M = pow(C, d, n)
    return M % 2
```

## 11. Why This Works

Because:

```text
Multiplying by 2 shifts bits
```

And:

```text
LSB tells if value crossed n/2 boundary
```

So each query gives:

```text
one bit of M
```

## 12. Complexity

```text
O(log n) queries
```

Very efficient.


## 13. When to Recognize

Look for:

```text
Oracle access
Server returns:
"even" / "odd"
"0" / "1"
```

Or hints like:

```text
"LSB"
"parity oracle"
"bit oracle"
```

## 14. Why Thhs Breaks RSA

Because:

```text
RSA is malleable (multiplicative)
```

And:

```text
Even tiny leakage (1 bit) → full plaintext recovery
```

## 15. Limitations

```text
Requires oracle access
Fails if padding (OAEP) is used
Needs many queries
```

## 16. Summary

```text
Multiply ciphertext by powers of 2
→ query LSB
→ shrink interval
→ recover M
```
