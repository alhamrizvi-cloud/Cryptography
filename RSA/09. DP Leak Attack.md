

# dp Leak Attack — Deep Explanation

## 1. Problem Setup

RSA parameters:

```text
n = p · q
e = public exponent
d = private exponent
```

Normally:

```text
e · d ≡ 1 mod φ(n)
```

---

## 2. What is leaked

Instead of full `d`, we are given:

```text
dp = d mod (p − 1)
```

This is used internally in CRT-based RSA for fast decryption.

---

## 3. Core Mathematical Derivation

From:

```text
d ≡ dp mod (p − 1)
```

So:

```text
d = dp + t(p − 1)
```

Multiply both sides by `e`:

```text
e·d = e·dp + t·e(p − 1)
```

But we know:

```text
e·d ≡ 1 mod (p − 1)
```

So:

```text
e·dp ≡ 1 mod (p − 1)
```

Which implies:

```text
e·dp − 1 = k(p − 1)
```

---

## 4. Rearranging

```text
p = (e·dp − 1)/k + 1
```

---

## 5. Key Insight

```text
k is a small integer
```

Why?

Because:

```text
e·dp − 1 < e(p − 1)
```

So:

```text
k < e
```

This makes brute force feasible.

---

## 6. Attack Strategy

We compute:

```text
A = e·dp − 1
```

Then loop:

```text
for k in [1, e):
    if A % k == 0:
        p = A/k + 1
        check if p divides n
```

---

## 7. Full Attack Flow

```text
Recover p
→ compute q = n / p
→ compute φ(n)
→ compute d
→ decrypt
```

---

## 8. Python Script (Clean)

```python
from sympy import mod_inverse

n = ...
e = ...
dp = ...
C = ...

A = e * dp - 1

for k in range(1, e):
    if A % k != 0:
        continue

    p = A // k + 1

    if p > 1 and n % p == 0:
        q = n // p
        break

phi = (p - 1) * (q - 1)
d = mod_inverse(e, phi)

M = pow(C, d, n)

print("p =", p)
print("q =", q)
print("d =", d)
print("M =", M)
```

---

## 9. Important Edge Cases

### Case 1 — Wrong k values

Most `k` values will fail:

```text
A % k ≠ 0
```

Skip them.

---

### Case 2 — False positives

Even if divisible:

```text
p = A/k + 1
```

You must check:

```text
n % p == 0
```

---

### Case 3 — dp corresponds to q

Sometimes given:

```text
dq = d mod (q − 1)
```

Same logic:

```text
q = (e·dq − 1)/k + 1
```

---

## 10. When to Recognize This Attack

Look for:

```text
dp or dq explicitly given
"CRT parameters leaked"
fast RSA hints
```

---

## 11. Why This Breaks RSA Completely

Because:

```text
Knowing p → factoring n becomes trivial
```

Then:

```text
Everything follows normally
```

## 12. Worked Example (Manual)

Given:

```text
n = 55
e = 3
dp = 3
```

Step 1:

```text
A = 3·3 − 1 = 8
```

Step 2:

```text
k = 1 → p = 9 (invalid)
k = 2 → p = 5 (valid)
```

Check:

```text
55 % 5 = 0
```

So:

```text
p = 5, q = 11
```

```text
O(e) loop
```
