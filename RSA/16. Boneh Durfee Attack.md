# Boneh–Durfee Attack (Very Small d)

## 1. Overview

This is a stronger version of Wiener’s attack.

```text id="c9s6r0"
Wiener works if: d < n^0.25
Boneh–Durfee works if: d < n^0.292
```

So it breaks RSA even when:

```text id="y9u2d7"
d is not extremely small, but still “too small”
```

---

## 2. Problem Setup

RSA:

```text id="q7q3iw"
n = p · q
e · d ≡ 1 mod φ(n)
```

Rewrite:

```text id="l3rq7p"
e·d − 1 = k·φ(n)
```

---

## 3. Core Idea

We want to solve:

```text id="pjb2nq"
e·d − k·φ(n) = 1
```

But we don’t know `φ(n)`.

Use approximation:

```text id="v6f7k9"
φ(n) ≈ n − (p + q) + 1
```

Since:

```text id="f9d1hc"
p + q is small compared to n
```

---

## 4. Key Transformation

Let:

```text id="7f3f4w"
φ(n) = n − x
```

Then:

```text id="k8n4zv"
e·d − k(n − x) = 1
```

Rearrange:

```text id="0p8c0g"
e·d − k·n + k·x = 1
```

---

## 5. Unknowns

```text id="1e1gq6"
d is small
k is small
x is small (≈ p + q)
```

So we now have a **multivariable small root problem**.

---

## 6. Key Insight

```text id="3l0w9m"
Treat equation as polynomial
Solve using lattice reduction
```

Using:

```text id="i1v7o3"
LLL algorithm + Coppersmith method
```

---

## 7. Why It Works

```text id="5tq2c6"
d is small → solution lies in small region
```

Lattice finds:

```text id="l7w2va"
small integer solutions to polynomial equations
```

---

## 8. Attack Strategy

1. Build polynomial:

```text id="qk3n0l"
f(d, k, x) = e·d − k(n − x) − 1
```

2. Construct lattice

3. Apply LLL

4. Recover small roots

5. Extract `d`
