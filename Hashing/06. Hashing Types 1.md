## Key Understanding Before We Start

### 1. Not all hashes are for security

* Cryptographic hashes → MD5, SHA family, BLAKE2, Whirlpool
* Non-cryptographic hashes → CRC32, MurmurHash

### 2. Security depends on design purpose

* Cryptographic hashes → resist attacks
* Non-cryptographic hashes → optimized for speed, not safety


### 3. CTF mindset reminder

When you encounter a hash:

* If it’s cryptographic but weak → try brute force or collision thinking
* If it’s modern (SHA-2, SHA-3, BLAKE2) → look for input weakness, not algorithm break
* If it’s non-crypto → expect manipulation or reversal
