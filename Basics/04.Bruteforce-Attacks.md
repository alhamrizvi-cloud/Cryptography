
## What is a Brute Force Attack?

A brute force attack is a method where an attacker tries **all possible combinations** of passwords or keys until the correct one is found.

Simple idea:
**Try everything until something works**

---

## When is it Used?

* When no other easier vulnerability is found
* When passwords or keys are **weak or short**
* In CTF challenges (very common)
* When attacker has access to login or encrypted data

---

## Why Does it Work?

* Computers can try **millions of guesses per second**
* Humans often use **weak passwords**
* Small key size means fewer combinations

If enough time is given, brute force can break anything in theory.

---

## Simple Example

Password = `123`

Possible attempts:

```
000
001
002
...
123 (found)
```

---

## Types of Brute Force

* **Simple Brute Force** → Try all combinations
* **Dictionary Attack** → Try common passwords
* **Hybrid Attack** → Words + numbers (example: admin123)

---

## Common Tools

* **Hydra**
  Fast tool for brute forcing login services like SSH, FTP, HTTP

* **John the Ripper**
  Used to crack hashed passwords

* **Hashcat**
  Very fast password cracking using GPU

* **Burp Suite**
  Used for web-based brute force attacks using Intruder

---

## Real-World Use

* Password cracking
* Penetration testing
* CTF challenges
* Breaking weak encryption

---

## How to Prevent It

* Use **strong passwords** (long and complex)
* Enable **rate limiting**
* Use **CAPTCHA**
* Enable **2FA (Two-Factor Authentication)**

---

## One-Line Summary

**Brute Force = trying every possible key or password until success**
