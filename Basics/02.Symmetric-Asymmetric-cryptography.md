## 🔐 Symmetric Encryption (Diagram)

```
        🔑 SAME KEY

Sender                          Receiver
  |                                |
  | --(Encrypt using KEY)-->       |
  |        Ciphertext              |
  | ----------------------------> |
  |                                |
  | <--(Decrypt using SAME KEY)--  |
  |        Plaintext               |
```

### 🧠 Idea:

* One **shared secret key**
* Both sender & receiver must **have the same key**

---

## 🔑 Asymmetric Encryption (Diagram)

```
        🔓 Public Key        🔐 Private Key

Sender                          Receiver
  |                                |
  | --(Encrypt using PUBLIC KEY)-> |
  |        Ciphertext              |
  | ----------------------------> |
  |                                |
  | <--(Decrypt using PRIVATE KEY)|
  |        Plaintext              |
```

### 🧠 Idea:

* Two keys:

  * **Public Key** → anyone can use
  * **Private Key** → kept secret by receiver

---

## ⚖️ Simple Difference (Easy to Understand)

| Feature     | Symmetric 🔑                    | Asymmetric 🔐🔓     |
| ----------- | ------------------------------- | ------------------- |
| Keys used   | 1 key                           | 2 keys              |
| Speed       | Fast ⚡                          | Slow 🐢             |
| Security    | Less secure (key share problem) | More secure         |
| Key sharing | Difficult ❌                     | Easy ✅ (public key) |
| Example     | AES                             | RSA                 |

---

## 🧠 Super Simple Way to Remember

* **Symmetric = One key**
  → Like a **room key** 🔑 (same key to lock/unlock)

* **Asymmetric = Two keys**
  → Like a **mailbox** 📬

  * Anyone can **drop mail (public key)**
  * Only owner can **open (private key)**

---

## 🚀 One-Line Summary

* Symmetric → **fast but risky key sharing**
* Asymmetric → **secure but slower**
